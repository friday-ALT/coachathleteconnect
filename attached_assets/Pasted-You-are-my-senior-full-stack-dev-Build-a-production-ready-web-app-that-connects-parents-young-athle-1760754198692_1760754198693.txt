You are my senior full-stack dev. Build a production-ready web app that connects parents/young athletes with soccer coaches.

## Tech & Standards

- Tailwind CSS + shadcn/ui for UI
- Postgres (via Prisma ORM). Use a hosted DB (can default to Neon).

- Basic rate limiting on auth and search API routes.

## Core Concept
Two roles: **Athlete** and **Coach**. After login, user lands on a simple “role switch” page with two large square tabs: **Athlete** and **Coach**. Each role has its own onboarding form (slider/tab switch at the top of the signup page).

### Onboarding (single page with a slider at top)
- Toggle labels: “Athlete” | “Coach”.
- Athlete required fields: email, phone number, age, skill level (Beginner/Intermediate/Advanced), location (city/state + optional lat/lng), password.
- Coach required fields: name, location (city/state + optional lat/lng), short line of previous experience (max 160 chars), charging rate/price (per hour), email, phone number, password, profile picture.
- Validation + helpful errors. Save to DB.

## Data Model (Prisma)
- User { id, email (unique), phone, password_hash, role: 'ATHLETE' | 'COACH', createdAt }
- AthleteProfile { id, userId (fk), age: int, skillLevel: enum('Beginner','Intermediate','Advanced'), locationCity, locationState, lat, lng }
- CoachProfile { id, userId (fk), name, locationCity, locationState, lat, lng, experience: string, pricePerHour: int (in cents), ratingAvg: float, ratingCount: int, avatarUrl }
- TimeSlotRequest { id, athleteId (fk to User), coachId (fk to User), groupSize: int, desiredPosition: string, note: string?, status: enum('PENDING','ACCEPTED','DECLINED'), createdAt }
- Review { id, athleteId (fk), coachId (fk), rating: int (1–5), comment: string?, createdAt }

Migrations included.

## Pages & Routes (App Router)
- `/` → Login page with email/password + phone input. Link to “Create account” which opens onboarding with the Athlete/Coach slider.
- `/onboarding` → Slider at top to switch between Athlete/Coach forms (see fields above).
- `/dashboard` → Two large square tabs: **Athlete** | **Coach** (visible based on role; if user has both, show both tabs). 
   - Athlete tab → Search screen:
     - Search bar + filters: age (int), group size (int), skill level (enum).
     - Results list of coaches with: rating (out of 5), profile picture, coach name, location, price.
     - Each card has “Request Time Slot” button → modal form with fields: group size, position, optional note.
   - Coach tab → Requests screen:
     - Table/list of incoming **TimeSlotRequest** items with athlete name, age, position, profile picture, location, requested group size.
     - Actions: Accept / Decline (updates status).
- `/profile` → Edit (role-based fields). Upload avatar.
- `/reviews` → Athletes can leave a rating (1–5) + comment for a coach they requested and was accepted.
- `/api` routes:
  - `POST /api/auth/*` (if using NextAuth, set up Credentials provider).
  - `GET /api/coaches` with query params: `skillLevel`, `age`, `groupSize`, `q` (name/location search). Paginated, indexed queries.
  - `POST /api/requests` (create TimeSlotRequest).
  - `PATCH /api/requests/:id` (accept/decline; coach-only).
  - `POST /api/reviews` (athlete-only; validates relationship).
- Protect routes with middleware; role-based guards.
